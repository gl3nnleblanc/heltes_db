Thoroughly read the current state of the codebase and TODO.md. Understand what has been built, what the known gaps are, and where the implementation is shallow or brittle.

Then add 3–5 new tasks to TODO.md that represent genuinely valuable next steps. Good candidates include:

- Correctness gaps or edge cases the current implementation doesn't handle
- Performance bottlenecks visible from the current design
- Operability or observability improvements that would matter in practice
- Protocol extensions that follow naturally from what's already specced
- Anything that would make the existing TODO tasks easier or unblock them

Each new task should follow the existing format: a short bold title, a one-line description of the problem and approach, and a story point estimate in parentheses. Place each task under the most appropriate existing section, or create a new section if none fits.

Do not add tasks that are already covered by existing TODO entries. Do not add tasks that are purely cosmetic or trivially small. Be honest about difficulty — don't lowball estimates.
